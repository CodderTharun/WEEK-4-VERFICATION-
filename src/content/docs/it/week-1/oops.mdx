## Basics

### A. POP vs OOP

| Features | POP | OOP |
|---|---|---|
| Focus | Functions | Objects & Classes |
| Approach | Top-down | Bottom-up |
| Data Security | Low | High |
| Reusability | Limited | High |
| Example Languages | C, Pascal | Java, Python, C++ |


### B. Features of OOP

- Combining data and methods into a single unit (class) and restricting access.
- Hiding complex details and showing only essential features.
- Creating new classes from existing classes to reuse code.
- One function behaves in different ways based on input or object.
- Reusing properties and methods from another class.
- Protecting data from unauthorized access.


### C. Class and Object

#### 1. Class

A class is a user-defined blueprint or prototype from which objects are created.  
It defines the properties and behaviors common to all objects of one type.

**Example:**  
Class → Car  
Objects → BMW, Audi, Tesla

#### 2. Object

An object is an instance of a class that represents a real-world entity.  
Objects contain data and methods.


### D. Attributes and Methods

#### 1. Attributes

Attributes are variables/data members that store the state of an object.

#### 2. Methods

Methods are functions inside a class that define the behavior of an object.

### E. Message Passing

Message passing is the process by which objects communicate with each other by calling methods.

- One object sends a message to another object.
- The receiving object performs the requested action.
- Communication happens through method calls.

**Example:**  
A customer object sends a message to a bank object to withdraw money.

## Constructors and Keywords

### A. Constructor

A constructor is a special method used to create and initialize an object.  
It runs automatically when an object is created and sets the initial values of the object’s data.

#### 1. Features of Constructor

- Has the same name as the class.
- Called automatically when an object is created.
- Used to initialize object data members.
- Does not have a return type.
- Helps reduce repeated code.


#### 2. Real-Life Example

When a car is manufactured, its engine and color are initialized automatically.

### B. this Keyword

The `this` keyword refers to the current object of the class.  
It is used to access the current object's variables and methods inside the class.

#### 1. Uses of this Keyword

- Refers to the current class object.
- Distinguishes instance variables from local variables.
- Can be used to call current class methods.
- Can be used to invoke current class constructors.
- Helps improve code readability.

#### 2. Real-Life Example

A student saying “this book is mine” refers to the current student’s book.

#### 3. this Keyword Example

## Encapsulation

The idea of encapsulation is to bind the data members and methods into a single unit.  
It restricts direct access to data and allows access only through methods.

### A. Data Hiding

Data hiding means restricting direct access to variables using access modifiers like `private`.

### B. Getters and Setters

Getters and setters are public methods used to access and update private data members.

- Getter → Used to read data
- Setter → Used to modify data

### C. Controlled Access

Controlled access means data can only be accessed through authorized methods, improving security and integrity.

### D. Real-Life Example

You can use an ATM machine, but the internal details are hidden.


### E. Encapsulation Example

## Abstraction

Abstraction means showing only important details and hiding complex implementation.

### A. What vs How

- What an object does → Visible to user
- How it works internally → Hidden from user

### B. Abstract Class

An abstract class is a class that cannot be instantiated.  
It contains both abstract methods and normal methods.  
It is mainly used to achieve partial abstraction.

### C. Interface

An interface is a blueprint of a class that contains abstract methods.  
It is used to achieve full abstraction and multiple inheritance.

### D. Difference between Abstract Class and Interface

| Abstract Class | Interface |
|----------------|------------|
| Can have abstract and normal methods | Contains abstract methods |
| Supports partial abstraction | Supports full abstraction |
| Can have constructors | Cannot have constructors |
| A class extends an abstract class | A class implements an interface |
| Supports single inheritance | Supports multiple inheritance |


### E. Real-Life Example

You drive the car (what), but you do not know how the engine works internally (how).


### F. Abstraction Example

<Tabs>

  <TabItem label="Java">

```java
abstract class Animal {

    abstract void sound();
}

class Dog extends Animal {

    void sound() {
        System.out.println("Bark");
    }
}
```

  </TabItem>

  <TabItem label="C++">

```cpp
#include <iostream>
using namespace std;

class Animal {

public:

    virtual void sound() = 0;
};

class Dog : public Animal {

public:

    void sound() {
        cout << "Bark";
    }
};
```

  </TabItem>

  <TabItem label="Python">


```python
from abc import ABC, abstractmethod

class Animal(ABC):

    @abstractmethod
    def sound(self):
        pass

class Dog(Animal):

    def sound(self):
        print("Bark")
```

  </TabItem>

</Tabs>

The abstract method defines what to do, not how.

## Inheritance

Inheritance is a mechanism in which one class acquires the properties and behaviors of another class.

### A. Parent → Child Relationship

- Parent Class → The class whose properties are inherited.
- Child Class → The class that inherits properties from another class.

**Example:**
Animal → Parent Class  
Dog → Child Class

### B. IS-A Relationship

Inheritance represents an IS-A relationship.

**Examples:**
- Dog IS-A Animal
- Car IS-A Vehicle



### C. Types of Inheritance

There are 5 types of inheritance.

#### 1. Single Inheritance
In single inheritance, a subclass is derived from only one superclass.
It inherits properties and behavior from a single parent class.
It is also known as simple inheritance.

#### 2. Multiple Inheritance
In multiple inheritance, one class can have more than one superclass and inherit features from all parent classes.


#### 3. Hierarchical Inheritance
In hierarchical inheritance, more than one subclass is inherited from a single base class.
That means multiple derived classes are created from one base class.



#### 4. Multilevel Inheritance
Multilevel inheritance means a class is derived from another derived class, forming a chain of inheritance.



#### 5. Hybrid Inheritance
Hybrid inheritance occurs when two or more inheritance types are combined in one program.<br />
 - Example:
    - Multiple inheritance
    - Multilevel inheritance


### D. super Keyword

The `super` keyword refers to the parent class object.  
It is used to access parent class variables, methods, and constructors.

#### 1. Uses of super Keyword

- Refers to the parent class object.
- Accesses parent class variables.
- Calls parent class methods.
- Invokes parent class constructors.
- Helps in method overriding.


#### 2. Real-Life Example

A child calling their parent for help is similar to using the `super` keyword to access parent class members.

#### 3. super Keyword Example

  <TabItem label="Java">
