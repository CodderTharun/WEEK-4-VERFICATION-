## Object Orientedd Programming
## Procedural Oriented Programming (POP)

Procedural Oriented Programming (POP) is a programming paradigm that focuses on procedures (functions) as the main building blocks of a program.
It organizes code as a sequence of instructions that operate on data.

### A. Features of POP

- Programs are divided into small reusable functions.
- Start with a main problem → break it into smaller sub-problems (functions).
- Instructions are executed step-by-step.
- Data can be shared across functions using global variables.
- Data is often exposed to multiple functions.

### B. Advantages of POP

- Simple and easy to understand.
- Suitable for small programs.
- Code reuse through functions.
- Faster execution (less overhead than OOP).

### C. Disadvantages of POP

- Difficult to manage large programs.
- Data is not secure (no encapsulation).
- Hard to modify and debug complex systems.
- No real-world modeling.

---

## Object Oriented Programming (OOP)

OOPS is a programming approach that organizes code into objects, which contain data and methods, to make programs more structured, reusable, and easy to understand.

### A. Features of OOP

- Combining data and methods into a single unit (class) and restricting access.
- Hiding complex details and showing only essential features.
- Creating new classes from existing classes to reuse code.
- One function behaves in different ways based on input or object.
- Reusing properties and methods from another class.
- Protecting data from unauthorized access.


### B. Advantages of OOP

- Code can be reused using inheritance.
- The program is divided into small parts (classes), making it easy to manage.
- Errors can be fixed easily without affecting the whole program.
- Data is protected using encapsulation.
- Easy to extend and modify programs.
- Represents real-life objects clearly.


### C. Disadvantages of OOP

- Hard to understand for beginners.
- Requires extra memory for objects and classes.
- Compared to procedural programming in some cases.
- Extra effort needed for design (classes, objects).
- Simple problems do not need OOP structure.

### D. Characteristics of OOP

- Class
- Object
- Encapsulation
- Abstraction
- Polymorphism
- Inheritance

## Difference Between POP and OOP

| Features | POP | OOP |
|---|---|---|
| Focus | Functions | Objects & Classes |
| Approach | Top-down | Bottom-up |
| Data Security | Low | High |
| Reusability | Limited | High |

## Class and Object

### A. Class

A class is a user-defined blueprint or prototype from which objects are created.
It represents the set of properties or methods that are common to all objects of one type.
Using classes, you can create multiple objects with the same behavior instead of writing their code multiple times.

#### Important Points About Classes

- A class can have members defined as `public`, `private`, or `protected` to control accessibility.
- The class name should follow naming conventions, usually starting with a capital letter.
- The class body is enclosed with braces `{}` and defines data members and member functions.



### B. Object

An object is a basic unit of Object-Oriented Programming that represents real-life entities.
A typical C++ program creates many objects, which interact with each other by invoking methods.
The objects are what perform your code and are visible to the user.

####  Features of an Object

##### 1. State
It is represented by the attributes of an object.
It reflects the properties of an object.

##### 2. Behavior
It is represented by the methods of an object.
It reflects the response of an object to other objects.

##### 3. Identity
It is a unique name given to an object that enables it to interact with other objects.

##### 4. Method
A method is a collection of statements that perform some specific task and return the result to the caller.

## Encapsulation

The idea of encapsulation is to bind the data members and methods into a single unit.

### A. Features of Encapsulation

- Helps in better maintainability, readability, and usability.
- Avoids explicitly passing data members to methods.
- Maintains data integrity by validating values assigned to variables.
- Helps achieve abstraction.
- Hides implementation details and exposes only required functionalities.

- Restricts direct access to private variables.
- Protects sensitive data from unauthorized access.
- Internal implementation can be changed without affecting users.
- Allows validation and control over data.
- Encapsulated classes can be reused easily.
- Promotes organized and modular code.



### E. Disadvantages of Encapsulation

- Writing getter and setter methods increases code length.
- Accessing data through methods adds slight performance overhead.
- Too much restriction may limit flexibility.

## Abstraction

Abstraction is another key concept in Object-Oriented Programming.
It means showing only the important details and hiding the complex implementation.

#### Abstraction Focuses On

- What to do (functionality)
- Not how it is done (implementation)

### C. Advantages of Abstraction

- Focus only on required details.
- Makes code cleaner and easier to understand.
- Hides important implementation details.
- Common structure can be reused.
- Changes in implementation do not affect users.



### D. Disadvantages of Abstraction

- Requires proper planning.
- Abstract classes may feel confusing for beginners.
- Can increase complexity if overused.
- Indirect calls may slightly affect performance.

## Inheritance

Inheritance is a mechanism in which one class acquires the properties and behaviors of another class.
Inheritance allows:

- Code reuse
- Extension of existing classes
- Hierarchical classification



### A. Features of Inheritance

- One class can reuse properties of another class.
- Child classes reuse parent class features.
- Promotes code reusability.
- Represents real-world relationships.



### B. Inheritance Example

#### 1. Single Inheritance
In single inheritance, a subclass is derived from only one superclass.
It inherits properties and behavior from a single parent class.
It is also known as simple inheritance.

#### 2. Multiple Inheritance
In multiple inheritance, one class can have more than one superclass and inherit features from all parent classes.



#### 3. Hierarchical Inheritance
In hierarchical inheritance, more than one subclass is inherited from a single base class.
That means multiple derived classes are created from one base class.



#### 4. Multilevel Inheritance
Multilevel inheritance means a class is derived from another derived class, forming a chain of inheritance.



#### 5. Hybrid Inheritance
Hybrid inheritance occurs when two or more inheritance types are combined in one program.<br />
 - Example:
    - Multiple inheritance
    - Multilevel inheritance

### A. Compile-Time Polymorphism

Compile-time polymorphism is also known as:

- Early Binding
- Static Polymorphism

In compile-time polymorphism, the compiler determines how the function or operator will work depending on the context during compilation.

This type of polymorphism is achieved using:

- Function Overloading
- Operator Overloading

#### 2. Operator Overloading
Programming languages can provide operators with special meanings for particular data types.
This ability is known as Operator Overloading.
For example:
- The `+` operator is used to add integers.
- The `+` operator is also used to concatenate strings.
- The `<<` and `>>` operators are used as binary shift operators and also with input/output streams.
This is possible because of operator overloading.

### B. Runtime Polymorphism

It is also known as late binding and dynamic polymorphism, the function call in runtime polymorphism is resolved at runtime in contrast with compile time polymorphism, where the compiler determines which function call to bind at compilation. Runtime polymorphism is implemented using function overriding with virtual functions.


#### 1. Function Overriding

Function Overriding occurs when a derived class defines one or more member functions of the base class. That base function is said to be overridden. The base class function must be declared as a virtual function for runtime polymorphism to happen.


##### Function Overriding Example

#### 2. Advantages of Runtime Polymorphism

- Increases flexibility in programs.
- Supports method overriding.
- Enables dynamic method dispatch.
- Makes code reusable and extensible.



#### 3. Disadvantages of Runtime Polymorphism

- Slightly slower because method calls are resolved at runtime.
- Increases program complexity.
- Debugging may become difficult in large systems.
