## What is OS?
An Operating System (OS) is a system software that acts as an interface between the user and the computer hardware and manages all resources efficiently.

## OS Operations

Operating System (OS) operations are the core activities an operating system performs to manage hardware, software, and user interactions.

### A. Process Management

The OS creates, schedules, and terminates processes (instances of running programs). It provides mechanisms for process synchronization, communication, and deadlock handling.

### B. Memory Management

The OS allocates and manages the computer’s memory resources, ensuring efficient and secure memory usage by processes.

### C. File Systems

The OS manages files and directories, providing methods for organizing, storing, and accessing data on secondary storage devices.

### D. Input/Output (I/O) Management

The OS handles communication between the computer system and external devices, managing input and output operations to ensure efficient data transfer.

### E. Device Management

The OS interacts with device drivers to control and manage hardware devices, including input/output devices, storage devices, and network devices.

### F. User Interface

The OS provides a user-friendly interface for users to interact with the system, including command-line interfaces, graphical user interfaces (GUIs), and application programming interfaces (APIs) for software development.

### G. Security and Protection

The OS enforces security policies, authenticates users, and protects system resources from unauthorized access or malicious activities.

### H.Error Handling and Fault Tolerance

The OS detects and handles errors, ensuring system stability and providing fault tolerance mechanisms to recover from failures.

## Functions of Operating System

### A. Process Management

A process is a program in execution. A process needs certain resources, including CPU time, memory, files, and I/O devices, to accomplish its task.

The operating system is responsible for the following activities in connection with process management:

- Process creation and deletion
- Process suspension and resumption
- Provision of mechanisms for:
  - Process synchronization
  - Process communication


### B. Main-Memory Management

Memory is a large array of words or bytes, each with its own address. It is a repository of quickly accessible data shared by the CPU and I/O devices.
Main memory is a volatile storage device. It loses its contents in the case of system failure.
The operating system is responsible for the following activities in connection with memory management:
- Keeping track of which parts of memory are currently being used and by whom
- Deciding which processes to load when memory space becomes available
- Allocating and de-allocating memory space as needed


### C. File Management

A file is a collection of related information defined by its creator. Commonly, files represent programs (both source and object forms) and data.
The operating system is responsible for the following activities in connection with file management:
- File creation and deletion
- Directory creation and deletion
- Support of primitives for manipulating files and directories
- Mapping files onto secondary storage
- File backup on stable (nonvolatile) storage media


### D. I/O System Management

The I/O system consists of:
- A buffer-caching system
- A general device-driver interface
- Drivers for specific hardware devices



### E. Secondary-Storage Management

Since main memory (primary storage) is volatile and too small to accommodate all data and programs permanently, the computer system must provide secondary storage to back up main memory.
Most modern computer systems use disks as the principal online storage medium for both programs and data.
The operating system is responsible for the following activities in connection with disk management:
- Free space management
- Storage allocation
- Disk scheduling

## Networking (Distributed Systems)
A distributed system is a collection of processors that do not share memory or a clock. Each processor has its own local memory.
The processors in the system are connected through a communication network. Communication takes place using a protocol.
A distributed system provides user access to various system resources.
Access to a shared resource allows:
- Computation speed-up
- Increased data availability
- Enhanced reliability

---

## Protection System
Protection refers to a mechanism for controlling access by programs, processes, or users to both system and user resources.
The protection mechanism must:
- Distinguish between authorized and unauthorized usage
- Specify the controls to be imposed
- Provide a means of enforcement

---

## Command-Interpreter System
Many commands are given to the operating system by control statements which deal with:
- Process creation and management
- I/O handling
- Secondary-storage management
- Main-memory management
- File-system access
- Protection
- Networking
The program that reads and interprets control statements is called:
- Command-line interpreter
- Shell (in UNIX)
Its function is to get and execute the next command statement.

## Evolution of OS

### A. Mainframe Systems

- Reduce setup time by batching similar jobs
- Automatic job sequencing transfers control from one job to another
- First rudimentary operating system
- Resident monitor:
  - Initial control in monitor
  - Control transfers to job
  - When job completes, control transfers back to monitor


### B. Batch Processing Operating System

This type of OS accepts more than one job and batches/groups them according to similar requirements.
Features:
- Jobs are grouped by a computer operator
- Jobs are executed when the computer becomes available
- Output is gradually sent back to the user
- Allowed only one program at a time
- Schedules jobs according to priority and required resources

### C. Multiprogramming Operating System

This type of OS executes more than one job simultaneously using a single processor.
#### 1. Features

- Increases CPU utilization
- Jobs are stored in a job pool on disk
- OS loads jobs into main memory and begins execution
- If one job waits for I/O, CPU switches to another job
- CPU remains busy as long as at least one job is ready

#### 2. Uses

- Job scheduling
- CPU scheduling

### D. Time-Sharing / Multitasking Operating Systems

Time-sharing is a logical extension of multiprogramming.
#### 1. Features
- Faster switching between jobs
- Multiple users can share the system simultaneously
- Users can interact with jobs while running
- Uses virtual memory for effective memory utilization

#### 2. Uses
- CPU scheduling
- Memory management
- Disk management
- Security management

#### 3. Examples
- CTSS
- MULTICS
- CAL
- UNIX

### E. Multiprocessor Operating Systems

Multiprocessor operating systems are also known as parallel OS or tightly coupled OS.
These systems have more than one processor in close communication, sharing:
- Computer bus
- Clock
- Memory
- Peripheral devices

#### 1. Advantages

##### a. Increased Throughput
By increasing the number of processors, the system performs more work in less time.

##### b. Economy of Scale
Processors can share peripherals, mass storage, and power supplies.

##### c. Increased Reliability
Failure of one processor does not halt the system completely.
The ability to continue service proportional to surviving hardware is called **graceful degradation**.
Systems designed for graceful degradation are called **fault tolerant** systems.

#### 2. Types

1. Symmetric multiprocessing system
2. Asymmetric multiprocessing system

##### a. Symmetric Multiprocessing

Each processor runs an identical copy of the OS.

##### b. Asymmetric Multiprocessing

One master processor controls slave processors.


### F. Distributed Operating Systems

In distributed systems:
- Machines are connected in a network
- Each machine has its own processor and local memory
- All operating systems cooperate to manage network resources

#### 1. Categories

1. Client-Server systems
2. Peer-to-Peer systems

#### 2. Advantages

- Resource sharing
- Computation speed-up
- Reliability
- Communication

#### 3. Networking Infrastructure

- LAN
- WAN


### G. Desktop Systems / Personal Computer Systems

The PC operating system is designed for maximizing user convenience and responsiveness.

#### 1. Features

- Neither multi-user nor multitasking
- Includes Microsoft Windows and Apple Macintosh
- Supports virtual memory and multitasking in modern systems


### H. Real-Time Operating Systems (RTOS)

A real-time operating system (RTOS) is intended for applications with fixed deadlines.

#### 1. Applications

- Embedded systems
- Automobile engine controllers
- Industrial robots
- Spacecraft
- Industrial control systems

#### 2. Types

1. Hard real-time systems
2. Soft real-time systems

##### a. Hard Real-Time System

Guarantees completion of critical tasks on time.

##### b. Soft Real-Time System

Critical tasks get higher priority but timing guarantees are less strict.

The shell is the outermost layer of the operating system and handles user interaction.

The kernel is the core component of the operating system and acts as the primary interface between the OS and hardware.

## Kernel in Operating System

A kernel is the core part of an operating system.
It acts as a bridge between software applications and computer hardware.
The kernel manages:

- CPU
- Memory
- Devices

It handles:

- Running programs
- Accessing files
- Connecting devices



### Types of Kernels

#### 1. Monolithic Kernel

All OS services run in kernel space.

##### a. Features

- Fast
- Less fault isolation

##### b. Examples

- UNIX
- Linux
- Open VMS
- XTS-400


#### 2. Microkernel

Minimal kernel functionality with most services moved to user space.

##### a. Features

- Better reliability
- More overhead

##### b. Examples

- Minix 3
- Mach 3.0


#### 3. Hybrid Kernel

Combines monolithic and microkernel concepts.

##### Examples

- Windows NT family
- macOS / XNU
- ReactOS
- Haiku OS


#### 4. Nanokernel

Extremely minimal kernel providing only hardware abstraction.

##### Examples

- Nemesis
- MIT Exokernel projects


#### 5. Exokernel

The OS handles only protection and gives applications direct access to hardware resources.

##### Examples

- MIT Exokernel
- Xok
- ExOS


### Functions of Kernel

#### 1. Process Management

Scheduling and execution of processes.

#### 2. Memory Management

- Allocation and deallocation
- Virtual memory handling
- Memory protection

#### 3. Device Management

Managing input/output devices and drivers.

#### 4. File System Management

Managing file operations and file system interfaces.

#### 5. Resource Management

Managing:

- CPU time
- Disk space
- Network bandwidth

#### 6. Security and Access Control

Enforcing permissions and authentication.

#### 7. Inter-Process Communication

Providing:
- Message passing
- Shared memory



### Working of Kernel

- Kernel is loaded into memory during boot
- Operates in privileged kernel mode
- Applications use system calls to request services
- Kernel switches from user mode to kernel mode
- Executes requested operations
- Returns results to user space
- Performs context switching for multitasking
