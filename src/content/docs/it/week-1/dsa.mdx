# 1. Theory

**Definition:**  
An array is a linear data structure that stores elements of the same data type in contiguous memory locations.

---

# 2. Key Characteristics of Arrays

| Characteristics | Explanation |
| --- | --- |
| Homogeneous Elements | All elements must be of the same data type |
| Contiguous Memory Allocation | Elements are stored in continuous memory locations |
| Fixed Size | Size is defined at declaration |
| Index-Based Access | Each element is accessed using an index |
| Random Access | Any element can be accessed instantly using index |
| Efficient Memory Usage | No extra memory overhead |
| Ordered Collection | Elements are stored in a fixed sequence/order |
| Static Nature | Cannot expand or shrink dynamically |
| Supports Multiple Dimensions | 1D, 2D, 3D arrays, etc. |

# Binary Search

Binary Search is a searching algorithm that operates on a sorted or monotonic search space, repeatedly dividing it into halves to find a target value or optimal answer in logarithmic time `O(log N)`.


---

# Conditions to Apply Binary Search Algorithm

- The data structure must be sorted.
- Access to any element of the data structure should take constant time.

---

# Binary Search Algorithm

1. Divide the search space into two halves by finding the middle index `mid`.
2. Compare the middle element with the key.
3. If the key is found at middle, terminate the process.
4. If the key is smaller than the middle element, search the left half.
5. If the key is larger than the middle element, search the right half.
6. Continue until the key is found or the search space is exhausted.

---

# Working of Binary Search

Consider:

```text
arr[] = {2, 5, 8, 12, 16, 23, 38, 56, 72, 91}

target = 23
```

---

# How to Implement Binary Search?

Binary Search can be implemented in two ways:

- Iterative Binary Search
- Recursive Binary Search

---

# Iterative Binary Search

Time Complexity: `O(log n)`  
Auxiliary Space: `O(1)`

Here we use a `while` loop to repeatedly divide the search space.

---
# Iterative Binary Search

<Tabs>
  <TabItem label="C++">

```cpp
#include <iostream>
#include <vector>
using namespace std;

int binarySearch(vector<int> &arr, int x) {

    int low = 0;
    int high = arr.size() - 1;

    while (low <= high) {

        int mid = low + (high - low) / 2;

        if (arr[mid] == x)
            return mid;

        if (arr[mid] < x)
            low = mid + 1;
        else
            high = mid - 1;
    }

    return -1;
}

int main() {

    vector<int> arr = {2, 3, 4, 10, 40};

    int x = 10;

    int result = binarySearch(arr, x);

    if (result == -1)
        cout << "Element is not present in array";
    else
        cout << "Element is present at index "
             << result;

    return 0;
}
```

# Recursive Binary Search

Recursive Binary Search uses recursion to divide the search space into halves repeatedly until the element is found.

Time Complexity: `O(log n)`  
Auxiliary Space: `O(log n)`

---

# Recursive Binary Search Algorithm

1. Find the middle index `mid`.
2. Compare the middle element with the target.
3. If the target is found, return the index.
4. If the target is smaller, search the left half recursively.
5. If the target is greater, search the right half recursively.
6. Continue until the element is found or the search space becomes empty.

---
# Recursive Binary Search

<Tabs>

```cpp
#include <iostream>
#include <vector>
using namespace std;

int binarySearch(vector<int>& arr,
                 int low,
                 int high,
                 int x) {

    if (high >= low) {

        int mid = low + (high - low) / 2;

        if (arr[mid] == x)
            return mid;

        if (arr[mid] > x)
            return binarySearch(
                arr,
                low,
                mid - 1,
                x
            );

        return binarySearch(
            arr,
            mid + 1,
            high,
            x
        );
    }

    return -1;
}

int main() {

    vector<int> arr = {2, 3, 4, 10, 40};

    int query = 10;

    int n = arr.size();

    int result = binarySearch(
        arr,
        0,
        n - 1,
        query
    );

    if (result == -1)
        cout << "Element is not present in array";
    else
        cout << "Element is present at index "
             << result;

    return 0;
}
```

# Merge Sort

Merge Sort is a popular sorting algorithm based on the Divide and Conquer approach.

It recursively divides the array into two halves, sorts them individually, and finally merges them together to form a sorted array.
# Working of Merge Sort

Merge Sort works using the Divide and Conquer technique.

## Steps Involved

### Divide

Divide the array recursively into two halves until each subarray contains only one element.

### Conquer

Sort each subarray individually.

### Merge

Merge the sorted subarrays back together in sorted order.

---

# Example of Merge Sort

Consider the array:

```text
[38, 27, 43, 10]
```

## Divide Phase

```text
[38, 27, 43, 10]
→ [38, 27] and [43, 10]

[38, 27]
→ [38] and [27]

[43, 10]
→ [43] and [10]
```

---

## Conquer Phase

```text
[38] is already sorted
[27] is already sorted
[43] is already sorted
[10] is already sorted
```

---

## Merge Phase

```text
Merge [38] and [27]
→ [27, 38]

Merge [43] and [10]
→ [10, 43]

Merge [27, 38] and [10, 43]
→ [10, 27, 38, 43]
```

---

# Final Sorted Array

```text
[10, 27, 38, 43]
```

```cpp
#include <iostream>
#include <vector>
using namespace std;

void merge(vector<int>& arr,
           int left,
           int mid,
           int right) {

    int n1 = mid - left + 1;
    int n2 = right - mid;

    vector<int> L(n1), R(n2);

    for (int i = 0; i < n1; i++)
        L[i] = arr[left + i];

    for (int j = 0; j < n2; j++)
        R[j] = arr[mid + 1 + j];

    int i = 0;
    int j = 0;
    int k = left;

    while (i < n1 && j < n2) {

        if (L[i] <= R[j]) {
            arr[k] = L[i];
            i++;
        }
        else {
            arr[k] = R[j];
            j++;
        }

        k++;
    }

    while (i < n1) {
        arr[k] = L[i];
        i++;
        k++;
    }

    while (j < n2) {
        arr[k] = R[j];
        j++;
        k++;
    }
}

void mergeSort(vector<int>& arr,
               int left,
               int right) {

    if (left >= right)
        return;

    int mid = left + (right - left) / 2;

    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);

    merge(arr, left, mid, right);
}

int main() {

    vector<int> arr = {38, 27, 43, 10};

    int n = arr.size();

    mergeSort(arr, 0, n - 1);

    for (int i = 0; i < arr.size(); i++)
        cout << arr[i] << " ";

    cout << endl;

    return 0;
}
```

# Recurrence Relation of Merge Sort

```text
T(n) = Θ(1),                    if n = 1

T(n) = 2T(n/2) + Θ(n),         if n > 1
```

Where:

- `2T(n/2)` → Recursive calls for two halves
- `Θ(n)` → Time taken to merge subarrays
Where:

- `T(n)` represents the total time taken by the algorithm to sort an array of size `n`.
- `2T(n/2)` represents the time taken to recursively sort the two halves of the array.
- `O(n)` represents the time taken to merge the two sorted halves.

---

# Complexity Analysis of Merge Sort

## Time Complexity

| Case | Complexity |
| --- | --- |
| Best Case | O(n log n) |
| Average Case | O(n log n) |
| Worst Case | O(n log n) |

### Explanation

- **Best Case:** When the array is already sorted or nearly sorted.
- **Average Case:** When the array elements are randomly arranged.
- **Worst Case:** When the array is sorted in reverse order.

---

## Auxiliary Space

```text
O(n)
```

Additional memory is required for temporary arrays during merging.

# Quick Sort

Quick Sort is a Divide and Conquer sorting algorithm.

It selects an element as a **pivot** and partitions the array around the pivot such that:

- Elements smaller than the pivot appear on the left side.
- Elements greater than the pivot appear on the right side.

Finally, the pivot reaches its correct sorted position.

---

# Steps of Quick Sort

## 1. Choose a Pivot

Select a pivot element from the array.

Possible pivot choices:

- First element
- Last element
- Random element
- Median element

---

## 2. Partition the Array

Rearrange the array so that:

- Smaller elements go to the left
- Larger elements go to the right

---

## 3. Recursive Calls

Apply Quick Sort recursively on:

- Left subarray
- Right subarray

---

## 4. Base Case

Recursion stops when the subarray contains only one element.

A single element is already sorted.

---

# Choice of Pivot

Different strategies for selecting pivot:

| Pivot Type | Description |
| --- | --- |
| First Element | Simplest approach |
| Last Element | Commonly used |
| Random Pivot | Reduces chances of worst case |
| Median Pivot | Improves partition balance |
There are different strategies for selecting a pivot in Quick Sort.

- Always pick the first element
- Always pick the last element
- Pick a random element
- Pick the median element

---

# Pivot Selection Strategies

## First or Last Element as Pivot

This is the simplest approach.

However, it performs poorly when the array is already sorted because it leads to the worst-case complexity.

---

## Random Pivot

A random element is selected as pivot.

This reduces the chance of consistently getting the worst-case partition.

---

## Median Pivot

The median element is selected as the pivot.

This usually gives balanced partitions and improves time complexity.

However, finding the median itself adds extra overhead.

---

# Partition Algorithm

The core operation in Quick Sort is the `partition()` function.

Partitioning rearranges the array around the pivot.

All partition algorithms work in:

```text
O(n)
```

time complexity.

---

# Types of Partition Algorithms

| Partition Method | Description |
| --- | --- |
| Naive Partition | Uses extra temporary array |
| Lomuto Partition | Simple and widely used |
| Hoare Partition | Faster and more efficient |

---

# Naive Partition

- Creates a temporary copy of the array.
- Places smaller elements first.
- Places larger elements later.
- Requires extra space:

```text
O(n)
```

---

# Lomuto Partition

This is the partition method used most commonly in learning Quick Sort.

## Working Principle

- Keep track of index `i`
- Traverse array using index `j`
- Swap elements smaller than pivot toward the left side

It is simple and easy to implement.

---

# Hoare Partition

Hoare Partition is faster than Lomuto Partition.

## Working Principle

- Traverse from both ends
- Swap misplaced elements
- Continue until partition is complete

It performs fewer swaps and is more efficient in practice.

---

# Working of Quick Sort

1. Select a pivot.
2. Partition the array.
3. Recursively apply Quick Sort on left subarray.
4. Recursively apply Quick Sort on right subarray.
5. Continue until subarrays contain only one element.

```cpp
#include <iostream>
#include <vector>
using namespace std;

int partition(vector<int>& arr,
              int low,
              int high) {

    int pivot = arr[high];

    int i = low - 1;

    for (int j = low;
         j <= high - 1;
         j++) {

        if (arr[j] < pivot) {

            i++;

            swap(arr[i], arr[j]);
        }
    }

    swap(arr[i + 1], arr[high]);

    return i + 1;
}

void quickSort(vector<int>& arr,
               int low,
               int high) {

    if (low < high) {

        int pi = partition(
            arr,
            low,
            high
        );

        quickSort(arr, low, pi - 1);

        quickSort(arr, pi + 1, high);
    }
}

int main() {

    vector<int> arr = {
        10, 7, 8, 9, 1, 5
    };

    int n = arr.size();

    quickSort(arr, 0, n - 1);

    for (int i = 0; i < n; i++) {
        cout << arr[i] << " ";
    }

    return 0;
}
```

```java
class GfG {

    static int partition(
        int[] arr,
        int low,
        int high
    ) {

        int pivot = arr[high];

        int i = low - 1;

        for (int j = low;
             j <= high - 1;

            if (arr[j] < pivot) {

                i++;

                swap(arr, i, j);
            }

        swap(arr, i + 1, high);

        int i,
```

# Maximum Subarray Sum - Kadane's Algorithm

Given an integer array `arr[]`, find the subarray with the maximum possible sum.

A subarray is a continuous part of the array.

---

# Examples

## Example 1

```text
Input:
arr[] = [2, 3, -8, 7, -1, 2, 3]

Output:
11
```

Explanation:

```text
Subarray = [7, -1, 2, 3]
Sum = 11
```

---

## Example 2

```text
Input:
arr[] = [-2, -4]

Output:
-2
```

---

## Example 3

```text
Input:
arr[] = [5, 4, 1, 7, 8]

Output:
25
```

## Idea

```cpp
int maxSubarraySum(vector<int>& arr) {

    int res = arr[0];

    for (int i = 0; i < arr.size(); i++) {

        int currSum = 0;

        for (int j = i; j < arr.size(); j++) {
```

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int maxSubarraySum(vector<int>& arr) {

    int res = arr[0];

    int maxEnding = arr[0];

    for (int i = 1; i < arr.size(); i++) {

        maxEnding = max(
            arr[i],
            maxEnding + arr[i]
        );

        res = max(res, maxEnding);
    }

}
```

```c
    int res = arr[0];

    int maxEnding = arr[0];

    for (int i = 1; i < size; i++) {

        maxEnding =
            (maxEnding + arr[i] > arr[i])
            ? maxEnding + arr[i]
            : arr[i];

        res =

            ? res
```
