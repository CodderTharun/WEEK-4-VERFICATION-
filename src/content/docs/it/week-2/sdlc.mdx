## Introduction to Software Engineering

Software Engineering is the systematic application of engineering principles and methods for developing, testing, deploying, and maintaining software systems.

Software engineering helps developers:

- Build reliable software
- Reduce software complexity
- Improve software quality
- Reduce development cost
- Deliver scalable applications
- Maintain systems efficiently

Software engineering is used in:

- Banking systems
- E-commerce applications
- Mobile applications
- Operating systems
- Healthcare systems

## Software vs Program

| Program | Software |
| --- | --- |
| A set of instructions written to perform a task | A complete package including programs, tools, libraries and documentation |
| Smaller in scope | Larger in scope |
| Mainly focuses on execution | Focuses on development, testing, deployment and maintenance |
| Example: Calculator Program | Example: Windows Operating System |

A program is simply a set of instructions written in a programming language.

Software is broader and includes:

- Programs
- Databases
- APIs
- Documentation
- Libraries

## Software Engineering Challenges

Software engineering faces several real-world challenges.

### A. Changing Requirements

Customer requirements may change during development due to:

- Market changes
- Customer feedback
- Technology updates

**Example:**  
An E-commerce application may initially support:

- Cash payments

Later it may require:

- UPI integration
- Wallet payments
- Credit card support

### B. Time Constraints

Projects must be completed within deadlines.

Late delivery can cause:

- Financial loss
- Customer dissatisfaction

### C. Budget Constraints

Projects must remain within allocated budgets.

Poor planning increases:

- Development cost
- Maintenance cost

### D. Software Complexity

Modern applications contain:

- APIs
- Databases
- Cloud systems
- Security layers

Managing such systems becomes difficult.

### E. Security Risks

Applications must protect:

- User data
- Transactions
- Authentication systems

### F. Maintenance Problems

After deployment, software requires:

- Bug fixing
- Updates
- Performance improvements

## Software Development Life Cycle (SDLC)

Software Development Life Cycle (SDLC) is a structured process used for designing, developing, testing, deploying and maintaining software systems.

SDLC ensures:

- Better software quality
- Better project management
- Easier maintenance
- Structured workflow
- Customer satisfaction

## Goals of SDLC

SDLC mainly focuses on:

- Efficiency
- Effectiveness
- Productivity

### A. Efficiency

Efficiency means:

**Build the system RIGHT.**

Focus areas include:

- Producing quality code
- Reducing errors
- Reducing rework

**Example:**  
Windows 98 experienced frequent crashes whereas Windows 10 became more stable and optimized because of improved software engineering practices.

### B. Effectiveness

Effectiveness means:

**Build the RIGHT system.**

Focus areas include:

- Customer satisfaction
- Correct functionalities
- Requirement fulfillment

**Example:**  
Tesla continuously updates vehicle software according to customer feedback.

### C. Productivity

Productivity focuses on:

- Faster development
- Better project management
- Reduced development cost

## Benefits of SDLC

SDLC provides several important benefits.

### A. Better Project Planning

Projects become more organized and manageable.

### B. Improved Software Quality

Testing and validation improve reliability.

### C. Reduced Development Risk

Risks are identified early.

### D. Easier Maintenance

Structured systems are easier to maintain and update.

### E. Better Communication

Teams communicate effectively using proper documentation.

### F. Reduced Rework Cost

Early error detection reduces correction cost.

## Stages of SDLC

The Software Development Life Cycle consists of five major phases.

1. Requirement Gathering & Analysis
2. Design Phase
3. Development Phase
4. Testing Phase

## Requirement Gathering & Analysis Phase

This phase focuses on collecting and analyzing customer requirements.

Communication occurs between:

- Customers
- Stakeholders
- End-users
- Project teams

Requirements may be:

- Functional requirements
- Non-functional requirements

### A. Functional Requirements

Functional requirements describe:

- What the system should do
- Features and functionalities

**Example:**  
For an Online Shopping System:

- User login
- Product search
- Add to cart
- Online payment

### B. Non-Functional Requirements

Non-functional requirements describe:

- Performance
- Security
- Reliability

**Example:**

- The system should support thousands of users.
- Transactions must be secure.

### C. Activities Performed

- Requirement collection
- Interviews
- Surveys
- Feasibility analysis
- Requirement documentation

### D. Feasibility Analysis

Feasibility analysis checks whether the project is practical.

#### 1. Types of Feasibility

##### a. Technical Feasibility

Checks whether required technology is available.

##### b. Financial Feasibility

Checks whether the project is affordable.

##### c. Operational Feasibility

Checks whether the organization can operate the system successfully.

### E. Software Requirement Specification (SRS)

All requirements are documented in:

**Software Requirement Specification (SRS)**

The SRS document acts as the foundation for future development.

## Design Phase

In this phase, software architecture and system design are prepared based on the SRS document.

### A. Activities Performed

- System architecture design
- Database design
- UML diagram creation
- High Level Design (HLD)
- Low Level Design (LLD)

### B. UML Diagrams

- Use Case Diagram
- Class Diagram
- Sequence Diagram
- Activity Diagram

### C. HLD and LLD

High Level Design (HLD) and Low Level Design (LLD) will be studied in detail in future weeks.

### D. Practical Example

For a Banking System:

- Database structure for accounts
- Login architecture
- Transaction workflow

## Development Phase

This phase involves actual software implementation and coding.

Developers convert design documents into working software.

### A. Activities Performed

- Writing source code
- Module development
- Unit testing
- Code reviews

### B. Practical Example

In an E-commerce Website:

- Cart module
- Payment gateway
- Product search system

## Testing Phase

Testing verifies whether the software works according to requirements.

The system is checked for:

- Bugs
- Errors
- Security issues
- Performance problems

### A. Types of Testing

- Unit Testing
- Integration Testing
- System Testing
- User Acceptance Testing (UAT)

### B. Activities Performed

- Bug identification
- Bug fixing
- Security testing
- Performance testing

### C. Practical Example

Testing:

- Login validation
- Product search
- Checkout process

## Deployment & Maintenance Phase

This is the final SDLC phase where software is released to users.

### A. Deployment

Software is:

- Installed
- Configured
- Released to users

### B. Maintenance Activities

#### 1. Corrective Maintenance

Fixing bugs and defects.

#### 2. Adaptive Maintenance

Updating software according to environmental changes.

#### 3. Perfective Maintenance

Improving features and performance.

#### 4. Preventive Maintenance

Preventing future software problems.

### C. Practical Example

Updating:

- Banking systems
- ERP software
- Mobile applications

## Process Models

Process models define how software development activities are organized and executed.

Important SDLC models include:

- Waterfall Model
- Iterative-Incremental Model
- V-Shaped Model
- RAD Model
- Spiral Model
- Agile Model

## Waterfall Model

The Waterfall Model is a linear and sequential software development model where each phase starts only after completion of the previous phase.

### A. Advantages

- Easy to understand
- Clear project stages
- Good for stable requirements

### B. Limitations

- Difficult to modify requirements
- Testing occurs late
- High rework cost

### C. Practical Example

Banking systems where requirements rarely change.

## Conclusion

Software Engineering provides structured methodologies for building reliable, scalable and maintainable software systems.

SDLC improves:

- Software quality
- Productivity
- Customer satisfaction
- Project management
- Risk handling

Different SDLC models are suitable for different project requirements, and selecting the correct model is important for successful software development.
