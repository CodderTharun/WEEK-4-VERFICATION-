## Hashing

Hashing is a technique used in data structures that efficiently stores and retrieves data in a way that allows for quick access.

Hashing involves mapping data to a specific index in a hash table (an array of items) using a hash function. It enables fast retrieval of information based on its key.

The great thing about hashing is, we can achieve all three operations (**search, insert and delete**) in **O(1)** time on average.

Hashing is mainly used to implement:

- Sets of distinct items (only keys)
- Dictionaries (key value pairs)

Here’s an example of hashing using the modulo method.

```math
H(x) = x \% 10
```

The hash function converts any large number into a smaller value between `0` and `9`.

---

### Applications of Hashing

Hashing is a technique that converts data of any size into a fixed-size value, known as a hash, which can be used for quick data retrieval and comparison.

---

#### 1. Data Integrity & Security

Hashing acts as a digital seal and lock, focusing on the verification and privacy of sensitive data.

#### Password Storage

Stores non-reversible "fingerprints" instead of plain text, allowing secure verification while keeping passwords hidden from attackers.

#### File Comparison and Message Digest

Generates unique signatures to verify file integrity, allowing users to instantly detect any corruption or malicious tampering.

#### Blockchain & Consensus

Links blocks via hashes to ensure permanent immutability, where any alteration breaks the chain and alerts the network.

#### Fraud Detection & Cybersecurity

Matches data patterns against known threat databases to identify and block malicious files the moment their hashes are detected.

---

### 2. Database & Search Optimization

This category leverages hashing to eliminate slow searches, jumping directly to the required data for high-performance computing.

#### Database Indexing

Enables near-instant record retrieval by using hashes as direct indices, bypassing slow sorting to speed up massive database queries.

#### Distinct Elements & Counting Frequencies

Tracks unique items and occurrences by using hash values as keys, providing a massive speed advantage over comparison methods.

#### Dictionaries & Associative Arrays

Facilitates immediate access to data via high-speed key mapping, ensuring lookups remain fast regardless of dataset size.

### Rabin-Karp Algorithm

Optimizes string-searching by hashing text segments to quickly filter out non-matches and focus only on potential pattern hits.

---

### 3. Network & System Infrastructure

Hashing optimizes traffic movement and resource usage to ensure minimal lag and disruption across complex systems.

#### Load Balancing

Uses consistent hashing to distribute traffic evenly across servers, ensuring stability and minimal data remapping during server changes.

#### Bloom Filters

A memory-efficient structure that trades a tiny margin of error for extreme speed when testing membership in massive data sets.

#### Network Routing

Hashes destination attributes to determine optimal data paths, minimizing latency and ensuring smooth transmission across global infrastructure.

#### Caching Mechanisms

Stores frequently accessed data under hash-based keys to reduce retrieval times and decrease server load by bypassing slow lookups.

---

### 4. Specialized Processing

These applications apply hashing to complex pattern recognition and low-level system organization.

## Image Processing

Employs perceptual hashing to find duplicate images by comparing structural features, identifying matches even after resizing or compression.

#### Symbol Tables

Maps code identifiers to memory locations during compilation, which is essential for accurate variable management and program execution.

#### Graphics & Grid Storage

Organizes graphical objects into spatial grids via hash functions to enable accelerated rendering and object management in 3D environments.

## Max Distance Between Two Occurrences

Given an array `arr[]`, the task is to find the maximum distance between two occurrences of any element.

If no element occurs twice, return `0`.

---

### Examples

#### Example 1

```text
Input: arr = [1, 1, 2, 2, 2, 1]
Output: 5
```

#### Explanation

- Distance for `1` is `5 - 0 = 5`
- Distance for `2` is `4 - 2 = 2`

So maximum distance is `5`.

---

#### Example 2

```text
Input: arr = [3, 2, 1, 2, 1, 4, 5, 8, 6, 7, 4, 2]
Output: 10
```

#### Explanation

- Max distance for `2` is `11 - 1 = 10`
- Max distance for `1` is `4 - 2 = 2`
- Max distance for `4` is `10 - 5 = 5`

---

#### Example 3

```text
Input: arr = [1, 2, 3, 6, 5, 4]
Output: 0
```

#### Explanation

No element occurs twice, so maximum distance is `0`.

---

### Naive Approach — Exploring All Pairs

- **Time Complexity:** `O(n²)`
- **Space Complexity:** `O(1)`

One by one, pick each element from the array and find its first and last occurrence in the array and take the difference between the first and last occurrence for maximum distance.

<Tabs>
  <TabItem label="C++">

```cpp
// C++ Program to find max distance between two occurrences
// in array by exploring all pairs

#include <iostream>
#include <vector>
using namespace std;

// function to find the maximum distance
int maxDistance(vector<int>& arr) {
    int res = 0;

    for (int i = 0; i < arr.size() - 1; i++) {
        for (int j = i + 1; j < arr.size(); j++) {

            // check if two elements are equal
            if (arr[i] == arr[j]) {

                // calculate the distance and update res
                res = max(res, j - i);
            }
        }
    }

    return res;
}

// Driver code
int main() {
    vector<int> arr = {1, 2, 4, 1, 3, 4, 2, 5, 6, 5};

    cout << maxDistance(arr) << endl;

    return 0;
}
```

## Using Hash Map or Dictionary

- **Time Complexity:** `O(n)`
- **Space Complexity:** `O(n)`

An efficient solution to this problem is to use hashing.

The idea is to:

1. Traverse the input array
2. Store the index of the first occurrence in a hash map
3. For every other occurrence:
   - Find the difference between the current index and the first index
   - Update the maximum distance if needed

---

<Tabs>
  <TabItem label="C++">

```cpp
// C++ Program to find max distance between two occurrences
// in array using hashing

#include <iostream>
#include <vector>
#include <unordered_map>
using namespace std;

int maxDistance(vector<int>& arr) {

    // Stores element to first index mapping
    unordered_map<int, int> mp;

    int res = 0;

    for (int i = 0; i < arr.size(); i++) {

        // If first occurrence
        if (mp.find(arr[i]) == mp.end())
            mp[arr[i]] = i;

        // Else update max distance
        else
            res = max(res, i - mp[arr[i]]);
    }

    return res;
}

int main() {

    vector<int> arr = {1, 1, 2, 2, 2, 1};

    cout << maxDistance(arr);

    return 0;
}
```
  </TabItem>

  <TabItem label="Java">

```java
// Java Program to find max distance between two occurrences
// in array using hashing

import java.util.*;

class GfG {

    static int maxDistance(int[] arr) {

        // Stores element to first index mapping
        HashMap<Integer, Integer> mp = new HashMap<>();

        int res = 0;

        for (int i = 0; i < arr.length; i++) {

            // If first occurrence
            if (!mp.containsKey(arr[i]))
                mp.put(arr[i], i);

            // Else update max distance
            else
                res = Math.max(res, i - mp.get(arr[i]));
        }

        return res;
    }

    public static void main(String[] args) {

        int[] arr = {1, 1, 2, 2, 2, 1};

        System.out.println(maxDistance(arr));
    }
}
```

## Using Hash Map or Dictionary — O(n) Time and O(n) Space

An efficient solution to this problem is to use hashing.

The idea is to:

1. Traverse the input array
2. Store the index of the first occurrence in a hash map
3. For every other occurrence:
   - Find the difference between the current index and the first index stored in the hash map
   - If the difference is greater than the current result, update the result

```cpp
        if (mp.find(arr[i]) == mp.end())
```

## String

Strings contain a small set of elements.

Unlike normal arrays, strings typically have a smaller set of items.

For example:

- Lowercase English alphabet has only `26` characters
- ASCII has only `256` characters

Strings are immutable in programming languages like:

- Java
- Python
- JavaScript
- C#

Many string problems can be optimized using the fact that the character set size is small.

Examples:

- Sorting can be done faster
- Counting frequencies becomes efficient
- Many interview questions are based on this concept

---

## Introduction to Strings

Strings are sequences of characters.

The differences between a character array and a string are:

- A string is terminated with a special character `'\0'`
- Strings are typically immutable in most programming languages

Examples of strings:

```text
"geeks"
"for"
"geeks"
"GeeksforGeeks"
"Geeks for Geeks"
"123Geeks"
"@123 Geeks"
```

## String Representation in Different Languages

### C

Strings are declared as character arrays or pointers and must end with a null character `\0` to indicate termination.

---

### C++

Supports both:

- C-style character arrays
- `std::string` class

The `std::string` class provides built-in functions for string manipulation.

---

### Java

---

### Python

Strings are immutable and can be declared using:

- Single quotes
- Double quotes
- Triple quotes

---

### JavaScript

Strings are immutable primitive data types and can be defined using:

- Single quotes
- Double quotes
- Template literals (backticks)

Template literals allow interpolation.

---

### C#

Uses the `string` keyword, which represents an immutable sequence of characters similar to Java.

### Important Note

There is no separate character type in:

- Python
- JavaScript

### 1. Length of String

The length of a string refers to the total number of characters present in it, including:

- Letters
- Digits
- Spaces
- Special characters

It is a fundamental property of strings and is often used in:

- Validation

---

### 2. Search a Character

Searching for a character in a string means finding the position where a specific character appears.

If the character appears multiple times, you may need:

- First occurrence
- Last occurrence
- All occurrences

---

### 3. Check for Substring

Checking for a substring means determining whether a smaller sequence of characters exists within a larger string.

A substring is a continuous part of a string.

This operation is common in:

- Text processing
- Search algorithms
- Data validation

---

### 3. Insert a Character

Inserting a character into a string means adding a new character at a specific position while maintaining the original order.

Since strings are immutable in many languages, this operation usually creates a new modified string.

---

### 4. Delete a Character

Deleting a character from a string means removing a specific character at a given position while keeping the remaining characters intact.

This generally creates a new string.

---

### 5. Check for Same Strings

Checking if two strings are the same means comparing them character by character.

They are considered identical if:

- Length is same
- Order is same
- Content is same

---

### 6. String Concatenation

String concatenation is the process of joining two or more strings together to form a single string.

Used in:

- Text processing
- Formatting messages
- Constructing file paths
- Dynamic content generation

---

### 7.  Reverse a String

Reversing a string means arranging its characters in the opposite order.

Commonly used in:

- Text manipulation
- Data encryption
- Coding challenges

---

### 8. Rotate a String

Rotating a string means shifting its characters to the left or right by a specified number of positions.

Characters moving past the boundary wrap around to the opposite side.

---

### 9. Check for Palindrome

Checking for a palindrome means determining whether a string reads the same forward and backward.

A palindrome remains unchanged when reversed.

Useful in:

- Text processing
- Algorithms
- Number theory

## String Matching Algorithms

String Matching Algorithms are used to find whether a pattern (substring) exists inside a main string (text).

They are widely used in:

- Search engines
- Text editors
- DNA sequence matching
- Compiler design
- Data filtering

---

### Types of String Matching Algorithms

| Algorithm | Idea |
|---|---|
| Naive String Matching | Compare characters one by one |
| Rabin-Karp Algorithm | Uses hashing |
| Knuth-Morris-Pratt (KMP) | Uses prefix table to avoid rechecking |
| Boyer-Moore Algorithm | Searches from right to left |
| Z Algorithm | Uses Z-array for pattern matching |

---

### Rabin-Karp Algorithm for Pattern Searching

Given two strings:

- `text` → the main text
- `pattern` → the substring pattern

Find all `0-based` starting indices where the pattern occurs in the text.

---

### Examples

#### Example 1

```text
Input:
text = "geeksforgeeks"
pattern = "geeks"

Output:
[0, 8]
```

### Explanation

The string `"geeks"` occurs at index `0` and `8`.

---

#### Example 2

```text
Input:
text = "aabaacaadaabaaba"
pattern = "aaba"

Output:
[0, 9, 12]
```

---

### How is Hash Value Calculated in Rabin-Karp?

The hash value is calculated using a **rolling hash function**.

It allows efficient hash updates by:

- Removing contribution of old character
- Adding contribution of new character

This avoids recalculating the entire hash from scratch.

---

### Hash Formula

For a string `s` of length `m`:

```math
hash(s) =
(s[0] \times d^{m-1} +
s[1] \times d^{m-2} + \dots +
s[m-1] \times d^0) \bmod q
```

---

### Where

- `s[i]` → ASCII value of character
- `d` → size of input alphabet (`256` for ASCII)
- `q` → prime number modulus (`101`, `1000000007`, etc.)

Using modulo:

- Prevents integer overflow
- Reduces hash collisions

---

### Rolling Hash Formula

Suppose we know the hash for substring:

```text
s[i-1 ... j-1]
```

We want hash for:

```text
s[i ... j]
```

---

### Previous Hash

```math
(s[i-1] \times d^{m-1} +
s[i] \times d^{m-2} + \dots +
s[j-1] \times d^0) \bmod q
```

---

### New Hash

```math
(s[i] \times d^{m-1} +
s[i+1] \times d^{m-2} + \dots +
s[j] \times d^0) \bmod q
```

---

### Rolling Hash Update

```math
hash(i, j) =
(d \times
(hash(i-1, j-1) - s[i-1] \times d^{m-1})
+ s[j]) \bmod q
```

---

## Steps in Rabin-Karp

1. Choose base `d`
2. Choose prime modulus `q`
3. Compute initial pattern hash
4. Compute initial text window hash
5. Slide pattern over text
6. Compare hashes
7. If hashes match:
   - Verify character-by-character

---

```cpp
#include <vector>
using namespace std;

// Function to search for all occurrences
// of 'pat' in 'txt' using Rabin-Karp

vector<int> search(const string &pat, const string &txt)
{
    int d = 256;
    int q = 101;

    int m = pat.length();
    int n = txt.length();

    int p = 0;
    int t = 0;

    int h = 1;

    vector<int> ans;

    // Precompute h = pow(d, m-1) % q
    for (int i = 0; i < m - 1; i++)
        h = (h * d) % q;

    // Initial hash values
    for (int i = 0; i < m; i++)
    {
        p = (d * p + pat[i]) % q;
        t = (d * t + txt[i]) % q;
    }

    // Slide pattern over text
    for (int i = 0; i <= n - m; i++)
    {
        if (p == t)
        {
            bool match = true;

            for (int j = 0; j < m; j++)
            {
                if (txt[i + j] != pat[j])
                {
                    match = false;
                    break;
                }
            }

            if (match)
                ans.push_back(i);
        }

        // Compute next window hash
        if (i < n - m)
        {
            t = (d * (t - txt[i] * h)
                 + txt[i + m]) % q;

            if (t < 0)
                t += q;
        }
    }

    return ans;
}

int main()
{
    string txt = "geeksforgeeks";
    string pat = "geeks";

    vector<int> res = search(pat, txt);

    for (auto it : res)
        cout << it << " ";

    return 0;
}
```

## KMP Algorithm

Given two strings:

- `txt`
- `pat`

Return all indices where `pat` occurs in `txt`.

---

### Examples

#### Example 1

```text
Input:
txt = "abcab"
pat = "ab"

Output:
[0, 3]
```

---

#### Example 2

```text
Input:
txt = "aabaacaadaabaaba"
pat = "aaba"

Output:
[0, 9, 12]
```

---

#### Why KMP?

Naive pattern matching becomes inefficient when the pattern contains repeating characters.

Examples:

```text
1) txt = "AAAAAAAAAAAAAAAAAB"
   pat = "AAAAB"

2) txt = "ABABABCABABABCABABABC"
   pat = "ABABAC"
```

KMP improves complexity to:

```text
O(n + m)
```

---

## Core Idea of KMP

When mismatch occurs after some matches:

- We already know certain characters will match
- So we avoid rechecking them

This optimization is achieved using:

```text
LPS Array
```

---

# What is LPS?

LPS = Longest Proper Prefix which is also Suffix.

---

# Example

For pattern:

```text
AAAA
```

LPS:

```text
[0, 1, 2, 3]
```

---

### Important Definitions

#### Proper Prefix

A prefix excluding the whole string.

Example:

```text
String = "abc"

Prefixes:
"", "a", "ab", "abc"

Proper Prefixes:
"", "a", "ab"
```

---

#### Suffixes

For `"abc"`:

```text
"", "c", "bc", "abc"
```

---

### LPS Formula

```text
lps[i] =
length of longest proper prefix
which is also suffix
for pat[0...i]
```

---

## Example 1 — Pattern "aabaaac"

| Index | LPS Value |
|---|---|
| 0 | 0 |
| 1 | 1 |
| 2 | 0 |
| 3 | 1 |
| 4 | 2 |
| 5 | 2 |
| 6 | 0 |

Final LPS:

```text
[0, 1, 0, 1, 2, 2, 0]
```

---

# Example 2 — Pattern "abcdabca"

Final LPS:

```text
[0, 0, 0, 0, 1, 2, 3, 1]
```

---

# More Examples of LPS

```text
Pattern = "AAAA"
LPS     = [0, 1, 2, 3]

Pattern = "ABCDE"
LPS     = [0, 0, 0, 0, 0]

Pattern = "AABAACAABAA"
LPS     = [0, 1, 0, 1, 2, 0, 1, 2, 3, 4, 5]
```

---

## Construction of LPS Array

#### Case 1

```text
pat[i] == pat[len]
```

Increase length.

---

## Case 2

```text
pat[i] != pat[len]
and len == 0
```

Set:

```text
lps[i] = 0
```

---

## Case 3

```text
pat[i] != pat[len]
and len > 0
```

Update:

```text
len = lps[len - 1]
```

```cpp
#include <iostream>
#include <string>
#include <vector>
using namespace std;

void constructLps(string &pat, vector<int> &lps)
{
    int len = 0;

    lps[0] = 0;

    int i = 1;

    while (i < pat.length())
    {
        if (pat[i] == pat[len])
        {
            len++;
            lps[i] = len;
            i++;
        }
        else
        {
            if (len != 0)
            {
                len = lps[len - 1];
            }
            else
            {
                lps[i] = 0;
                i++;
            }
        }
    }
}

{
    int n = txt.length();
    int m = pat.length();

    vector<int> lps(m);
    vector<int> res;

    int j = 0;

        if (txt[i] == pat[j])
        {
            i++;
            j++;

            if (j == m)
            {
                res.push_back(i - j);

                j = lps[j - 1];
            }
        }
        else
        {
            if (j != 0)
                j = lps[j - 1];
            else
                i++;
        }
    }

    return res;
}
```

### Introduction to Divide and Conquer Algorithm

Divide and Conquer is a problem-solving technique used to solve problems by:

1. Dividing the main problem into subproblems
2. Solving them individually
3. Merging results to solve the original problem

---

### When to Use Divide and Conquer

It works best when:

- Subproblems are independent

If subproblems overlap heavily:

- Dynamic Programming is preferred

---

### Steps in Divide and Conquer

Divide and Conquer consists of three phases:

1. Divide
2. Conquer
3. Merge

## Divide and Conquer Algorithm

The above diagram shows the working of **Merge Sort**, which is used for sorting.

---

### 1. Divide

Break down the original problem into smaller subproblems.

Each subproblem should represent a part of the overall problem.

The goal is to divide the problem until no further division is possible.

---

### Example in Merge Sort

In Merge Sort:

- The input array is divided into two halves

The divide step in Merge Sort is simple.

---

### Example in Quick Sort

In Quick Sort:

- The divide step is critical
- The array is partitioned around a pivot

---

## 2. Conquer

Solve each smaller subproblem individually.

If a subproblem is small enough (base case), solve it directly without further recursion.

The goal is to solve subproblems independently.

---

### Example in Merge Sort

The conquer step is:

- Sort the two halves individually

---

## 3. Merge

Combine the solved subproblems to get the final solution.

Once smaller subproblems are solved, recursively combine them to solve the larger problem.

---

### Example in Merge Sort

The merge step:

- Merges two sorted halves into one sorted array

The merge step is critical in Merge Sort.

---

### Example in Quick Sort

In Quick Sort:

- Merge step does nothing
- Both parts become sorted in-place

---

## Characteristics of Divide and Conquer Algorithm

Divide and Conquer works by:

1. Dividing the problem
2. Solving smaller parts
3. Combining solutions

---

### 1. Dividing the Problem

The problem is broken into smaller subproblems recursively until they become simple enough to solve directly.

---

### 2. Independence of Subproblems

Each subproblem should be independent.

Benefits:

- Parallel processing
- Concurrent execution
- Better efficiency

---

### 3. Conquering Each Subproblem

Each smaller problem is solved individually.

This may involve:

- Recursion
- Another algorithm

---

### 4. Combining Solutions

Solutions of smaller subproblems are combined to solve the original problem.

The combination step should be:

- Efficient
- Straightforward

---

## Examples of Divide and Conquer Algorithm

### 1. Merge Sort

Merge Sort divides the array into smaller subarrays, sorts them individually and merges them.

---

### 2. Quick Sort

Quick Sort:

- Chooses a pivot
- Places smaller elements to left
- Places greater elements to right
- Recursively sorts subarrays

---

# Applications of Divide and Conquer Algorithm

## [Approach 3] — Using Strassen's Method

Strassen’s algorithm originally applies to square matrices, but when adapted for multiplying an `n × m` matrix with an `m × q` matrix, the matrices are padded to form square matrices of size:

```math
s \times s
```

Where:

```math
s = \text{next power of two} \ge \max(n, m, q)
```

The algorithm then performs:

```text
7 recursive multiplications
```

on square blocks.

---

### Important Note

Strassen’s Method is often not preferred in practice because:

- High constant factors
- Naive multiplication is faster for small matrices
- Poor performance on sparse matrices
- Requires extra memory for recursive divisions
- Floating-point arithmetic accumulates more errors

---

```cpp
#include <iostream>
#include <vector>
#include <cmath>
using namespace std;

// Return next power of two
int nextPowerOfTwo(int n) {
    return pow(2, ceil(log2(n)));
}

// Resize matrix and fill extra space with zeros
vector<vector<int>> resizeMatrix(
    vector<vector<int>> &mat,
    int newR,
    int newC
) {
    vector<vector<int>> resized(
        newR,
        vector<int>(newC, 0)
    );

    for (int i = 0; i < mat.size(); ++i)
        for (int j = 0; j < mat[0].size(); ++j)
            resized[i][j] = mat[i][j];

    return resized;
}

// Matrix addition/subtraction
vector<vector<int>> add(
    vector<vector<int>> a,
    vector<vector<int>> b,
    int size,
    int sign = 1
) {
    vector<vector<int>> res(
        size,
        vector<int>(size)
    );

    for (int i = 0; i < size; i++)
        for (int j = 0; j < size; j++)
            res[i][j] =
                a[i][j] + sign * b[i][j];

    return res;
}

// Strassen multiplication
vector<vector<int>> strassen(
    vector<vector<int>> mat1,
    vector<vector<int>> mat2
) {
    int n = mat1.size();

    vector<vector<int>> res(
        n,
        vector<int>(n, 0)
    );

    // Base case
    if (n == 1) {
        res[0][0] =
            mat1[0][0] * mat2[0][0];

        return res;
    }

    int newSize = n / 2;

    vector<vector<int>> a11(newSize, vector<int>(newSize));
    vector<vector<int>> a12(newSize, vector<int>(newSize));
    vector<vector<int>> a21(newSize, vector<int>(newSize));
    vector<vector<int>> a22(newSize, vector<int>(newSize));

    vector<vector<int>> b11(newSize, vector<int>(newSize));
    vector<vector<int>> b12(newSize, vector<int>(newSize));
    vector<vector<int>> b21(newSize, vector<int>(newSize));
    vector<vector<int>> b22(newSize, vector<int>(newSize));

    // Split matrices
    for (int i = 0; i < newSize; i++) {

        for (int j = 0; j < newSize; j++) {

            a11[i][j] = mat1[i][j];
            a12[i][j] = mat1[i][j + newSize];
            a21[i][j] = mat1[i + newSize][j];
            a22[i][j] = mat1[i + newSize][j + newSize];

            b11[i][j] = mat2[i][j];
            b12[i][j] = mat2[i][j + newSize];
            b21[i][j] = mat2[i + newSize][j];
            b22[i][j] = mat2[i + newSize][j + newSize];
        }
    }

    // 7 recursive multiplications
    auto m1 = strassen(
        add(a11, a22, newSize),
        add(b11, b22, newSize)
    );

    auto m2 = strassen(
        add(a21, a22, newSize),
        b11
    );

    auto m3 = strassen(
        a11,
        add(b12, b22, newSize, -1)
    );

    auto m4 = strassen(
        a22,
        add(b21, b11, newSize, -1)
    );

    auto m5 = strassen(
        add(a11, a12, newSize),
        b22
    );

    auto m6 = strassen(
        add(a21, a11, newSize, -1),
        add(b11, b12, newSize)
    );

    auto m7 = strassen(
        add(a12, a22, newSize, -1),
        add(b21, b22, newSize)
    );

    // Final quadrants
    auto c11 =
        add(
            add(m1, m4, newSize),
            add(m7, m5, newSize, -1),
            newSize
        );

    auto c12 = add(m3, m5, newSize);

    auto c21 = add(m2, m4, newSize);

    auto c22 =
        add(
            add(m1, m3, newSize),
            add(m6, m2, newSize, -1),
            newSize
        );

    // Combine results
    for (int i = 0; i < newSize; i++) {

        for (int j = 0; j < newSize; j++) {

            res[i][j] = c11[i][j];
            res[i][j + newSize] = c12[i][j];

            res[i + newSize][j] = c21[i][j];
            res[i + newSize][j + newSize] =
                c22[i][j];
        }
    }

    return res;
}

// Multiply matrices
vector<vector<int>> multiply(
    vector<vector<int>> &mat1,
    vector<vector<int>> &mat2
) {
    int n = mat1.size();
    int m = mat1[0].size();
    int q = mat2[0].size();

    int size =
        nextPowerOfTwo(
            max(n, max(m, q))

    vector<vector<int>> cPad =

        vector<int>(q)
    );

    for (int i = 0; i < n; i++)
        for (int j = 0; j < q; j++)
            C[i][j] = cPad[i][j];

    return C;
}
```

```python
import math

# Return next power of two
def nextPowerOfTwo(n):

    return int(
        math.pow(
            2,
            math.ceil(math.log2(n))
        )
    )


# Resize matrix
def resizeMatrix(mat, newR, newC):

    resized = [
        [0 for _ in range(newC)]
        for _ in range(newR)
    ]

    for i in range(len(mat)):

        for j in range(len(mat[0])):

            resized[i][j] = mat[i][j]

    return resized


# Add/Subtract matrices
def add(a, b, size, sign=1):

    res = [
        [0 for _ in range(size)]
        for _ in range(size)
    ]

    for i in range(size):

        for j in range(size):

            res[i][j] = (
                a[i][j]
                + sign * b[i][j]
            )

    return res


# Strassen multiplication
def strassen(mat1, mat2):

    n = len(mat1)

    res = [
        [0 for _ in range(n)]
        for _ in range(n)
    ]

    # Base case
    if n == 1:

        res[0][0] = (
            mat1[0][0] * mat2[0][0]
        )

        return res

    newSize = n // 2

    a11 = [[0] * newSize for _ in range(newSize)]
    a12 = [[0] * newSize for _ in range(newSize)]
    a21 = [[0] * newSize for _ in range(newSize)]
    a22 = [[0] * newSize for _ in range(newSize)]

    b11 = [[0] * newSize for _ in range(newSize)]
    b12 = [[0] * newSize for _ in range(newSize)]
    b21 = [[0] * newSize for _ in range(newSize)]
    b22 = [[0] * newSize for _ in range(newSize)]

    # Split matrices
    for i in range(newSize):

        for j in range(newSize):

            a11[i][j] = mat1[i][j]
            a12[i][j] = mat1[i][j + newSize]

            a21[i][j] = mat1[i + newSize][j]
            a22[i][j] = mat1[i + newSize][j + newSize]

            b11[i][j] = mat2[i][j]
            b12[i][j] = mat2[i][j + newSize]

            b21[i][j] = mat2[i + newSize][j]
            b22[i][j] = mat2[i + newSize][j + newSize]

    # 7 recursive multiplications
    m1 = strassen(
        add(a11, a22, newSize),
        add(b11, b22, newSize)
    )

    m2 = strassen(
        add(a21, a22, newSize),
        b11
    )

    m3 = strassen(
        a11,
        add(b12, b22, newSize, -1)
    )

    m4 = strassen(
        a22,
        add(b21, b11, newSize, -1)
    )

    m5 = strassen(
        add(a11, a12, newSize),
        b22
    )

    m6 = strassen(
        add(a21, a11, newSize, -1),
        add(b11, b12, newSize)
    )

    m7 = strassen(
        add(a12, a22, newSize, -1),
        add(b21, b22, newSize)
    )

    # Result quadrants
    c11 = add(
        add(m1, m4, newSize),
        add(m7, m5, newSize, -1),
        newSize
    )

    c12 = add(m3, m5, newSize)

    c21 = add(m2, m4, newSize)

    c22 = add(
        add(m1, m3, newSize),
        add(m6, m2, newSize, -1),
        newSize
    )

    # Combine results
    for i in range(newSize):

        for j in range(newSize):

            res[i][j] = c11[i][j]

            res[i][j + newSize] = c12[i][j]

            res[i + newSize][j] = c21[i][j]

            res[i + newSize][j + newSize] = c22[i][j]

    return res

def multiply(mat1, mat2):

    n = len(mat1)
    m = len(mat1[0])
    q = len(mat2[0])

    size = nextPowerOfTwo(
        max(n, max(m, q))
    )

    aPad = resizeMatrix(mat1, size, size)

    bPad = resizeMatrix(mat2, size, size)

    cPad = strassen(aPad, bPad)

    C = [
        [0 for _ in range(q)]
        for _ in range(n)
    ]

    for i in range(n):

        for j in range(q):

            C[i][j] = cPad[i][j]

    return C
```
